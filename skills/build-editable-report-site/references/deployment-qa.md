# 部署与质量验证

## 本地验证

1. 严格校验无模板占位内容。
2. lint 和 build 通过。
3. 启动本地网站并打开首页。
4. 检查标题、核心结论、正文、图表、表格、来源和下载区域。
5. 进入编辑模式，直接修改一段文字并恢复。
6. 确认图表、指标、表格、SKU 和下载区域没有可编辑节点。
7. 空 Token 发布必须被拦截。
8. 刷新页面无运行错误。
9. 检查桌面端和窄屏布局。

## 发布验证

- GitHub 默认分支包含准确源代码。
- Sites 保存版本的 commit SHA 对应已推送源状态。
- 部署轮询到 `succeeded` 才算完成。
- 打开线上地址，重新检查页面加载和编辑边界。

## 必须披露

- 未用真实 Token 执行线上内容写入时，明确说明只验证了前端编辑、校验和发布代码。
- 公开站点的下载文件默认公开。
- 无法验证的浏览器、域名、权限或外部服务状态不得描述为已完成。
