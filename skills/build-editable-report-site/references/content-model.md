# 内容与数据模型

## 文件职责

### `content/report.json`

只保存可编辑叙事：

- 品牌与报告标题；
- 核心结论和解释；
- 章节标题、摘要、图表标题和解读；
- 异常值解释、机会、限制；
- 数据来源和计算口径说明；
- 附录与页脚文字。

### `data/report-data.json`

只保存锁定事实：

- 图表标签、数值、显示值和分母；
- 指标覆盖率；
- SKU、型号、价格、排名、销量和能力字段；
- 表头和表格单元格；
- 下载文件路径和类型。

### `public/runtime-config.json`

只保存公开运行配置：仓库、分支、内容路径和允许发布的 GitHub login。不得保存 Token、Secret、密码或私有 API Key。

## 页面块

V1 支持：

- `chart-group`：一个或多个柱形分布图；
- `metrics`：覆盖率或关键指标；
- `table`：下钻明细；
- `callout`：关键读数或限制；
- `details`：来源、口径和长表说明。

缺少新块类型时优先扩展通用渲染器，并保持内容/数据分离。不得把新报告的事实数组直接写入 React 组件。

## Markdown 映射

- “结论摘要”映射为 `findings`。
- 每条结论保留标题、解释、来源和目标章节。
- 正文一级主题映射为 `sections`。
- 图表标题和解读属于 content；图表行和数值属于 data。
- 过长但支撑结论的数据表放正文末尾或附录，不得完全省略。
