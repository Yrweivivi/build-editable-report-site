# 报告呈现规范

## 阅读结构

首页直接展示核心结论和关键证据，不显示内部生成规则、置信度控件或“查看论证”。正文保持扁平，避免传统 BI Dashboard 的卡片堆叠。

每条重要结论按以下结构：

1. 结论；
2. 具体解释；
3. 图表或关键证据；
4. 必要的 SKU 或表格下钻；
5. 数据来源和计算口径。

## 内容密度

- 以确认版 Markdown 为下限，不机械复制，但不得减少关键数字、判断和限制。
- 只做适合网页的轻微表述调整、短标题和层级优化。
- 没有数据价值的结论不强行总结。
- 异常值分析呈现异常本身、能够确认的事实、当前解释假设和验证方向；不要展示“发现异常必须下钻”等内部规则。

## 可编辑边界

可编辑：标题、结论、解释、章节说明、图表解读、机会、限制、来源说明。

锁定：图表数值、比例、分子分母、SKU、价格、排名、销量、数据表、下载路径。
